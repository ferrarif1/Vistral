# 产品需求文档（PRD，中文版）

## 产品概述
Vistral 是一个 AI-native 视觉模型平台，支持自然语言与附件驱动交互。平台在保持 RVision 核心业务逻辑的同时，采用两类系统角色（`user` / `admin`）并结合基于所有权与能力的权限模型来管理模型相关操作。

## 问题陈述
传统视觉模型平台多为仪表盘式交互，用户需要在复杂界面中跳转操作，沟通成本高、流程割裂，不利于快速完成视觉模型任务。

## 目标用户
1. **用户（User）**：通过对话与附件使用模型；在具备相应能力时，可创建、管理、重训自己拥有的模型。
2. **管理员（Admin）**：负责审批、审计、治理、用户管理与系统级运营。

## 核心用例

### 用例 1：视觉模型对话查询
- **执行者**：User
- **目标**：用自然语言获取视觉模型结果
- **流程**：
  1. 用户发起会话
  2. 上传视觉内容（图片/文档等）
  3. 输入自然语言问题
  4. 平台处理并返回结果
  5. 用户继续追问

### 用例 2：模型管理
- **执行者**：具备模型管理能力的 User
- **目标**：上传、配置并管理视觉模型
- **流程**：
  1. 进入模型管理界面
  2. 上传模型文件与配置
  3. 配置参数与约束
  4. 提交审批
  5. 观察模型使用与性能

### 用例 3：审批流程
- **执行者**：Admin
- **目标**：审核提交模型
- **流程**：
  1. 接收待审通知
  2. 审查合规与安全
  3. 测试功能
  4. 通过或拒绝
  5. 更新模型状态

## 访问模型澄清
- `model owner` 不是一级系统角色。
- 所有权通过资源关系表达（如 `models.owner_user_id`）。
- 额外管理权限通过轻量能力表达（如 `user.capabilities` 包含 `manage_models`）。

## 功能需求

### FR-001 对话界面
- 支持自然语言输入
- 维护多轮会话上下文
- 支持会话附件

### FR-002 文件管理
- 展示上传文件状态
- 支持会话内删除
- 文件在流程中持续可见

### FR-003 多步骤流程
- 显示进度指示
- 刷新后保留流程状态
- 支持中断恢复

### FR-004 参数控制
- 高级参数默认折叠
- 明确可配置高级项
- 保存参数可见性偏好

### FR-005 访问控制
- 系统仅有两类角色：`user` / `admin`
- 结合所有权与能力进行授权
- 对特权动作保留审计日志

### FR-006 模型生命周期
- 支持模型上传与版本化
- 支持训练流程
- 支持审批流
- 支持发布与弃用

### FR-007 边缘推理
- 支持边缘部署
- 监控边缘节点健康
- 合理路由推理请求

### FR-008 审计与合规
- 记录模型交互行为
- 跟踪模型变更与审批
- 支持生成合规报告

## 非功能需求
- 性能：简单请求 <2s，复杂处理 <10s
- 可用性：99.9% 可用，RTO <1h
- 安全：传输与存储加密，定期审计
- 可扩展：支持横向扩容与自动伸缩
