# 贡献指南（中文版）

## 贡献原则
- 保持 Vistral 的 AI-native 特性：对话 + 附件驱动；
- 保持三方角色语义一致（拥有者/用户/管理员）；
- 优先共享与复用，避免页面各自风格漂移；
- 涉及行为/数据/API 变化时，先更新 `docs/*` 合同。

## 标准流程
1. 先写简短计划；
2. 必要时先更新合同文档；
3. 实现最小闭环改动；
4. 执行与改动范围匹配的最小检查；
5. 说明风险与下一步建议。

## 最低检查要求
- 仅文档改动：
  - 校验链接有效性；
  - 校验术语一致性。
- 含代码改动：
  - 运行项目对应的 lint/test/typecheck；
  - 校验与 `docs/data-model.md`、`docs/api-contract.md` 对齐。

## PR 说明要求
每次提交应明确：
- 改了什么；
- 为什么改；
- 做了哪些验证；
- 剩余风险与后续计划。
