# 用户流程与过程文档（中文版）

## 概述
本文档定义 Vistral 关键用户流程，强调对话 + 附件驱动体验、步骤可视化和状态可恢复。

## 主要流程

### Flow 1：基础模型交互
**执行者**：User  
**目标**：通过对话获得视觉模型结果

1. 进入主会话界面
2. 系统展示欢迎信息与历史会话
3. 用户选择模型
4. 输入自然语言问题
5. 上传相关附件
6. 系统显示文件状态（uploading / processing / ready）
7. 提交请求
8. 系统返回结果
9. 用户继续会话或结束
10. 系统保存会话

### Flow 2：模型上传与发布
**执行者**：User（限自有/被授权模型范围）  
**目标**：提交新模型并进入审批

1. 进入模型管理区
2. 点击“创建新模型”
3. 展示带进度的多步骤向导
4. Step1 填写元数据
5. Step2 上传模型文件并显示状态
6. Step3 配置参数（高级参数默认折叠）
7. Step4 隐私与访问设置
8. 用户提交审批
9. 系统入队审核
10. 返回确认信息

### Flow 3：审批流程
**执行者**：Admin  
**目标**：审核提交模型

1. 接收待审通知
2. 打开审批队列
3. 查看待审优先级
4. 选择模型
5. 查看模型细节与文件
6. 功能验证
7. 通过或拒绝
8. 通过后发布并通知提交用户
9. 拒绝后反馈提交用户
10. 记录审计日志

### Flow 4：多步骤训练流程
**执行者**：User（限自有/被授权模型范围）  
**目标**：使用新数据重训模型

1. 选择已有模型
2. 打开训练选项
3. 点击“重训模型”
4. 展示顶部 stepper
5. Step1 选择数据集（带附件状态）
6. Step2 配置训练参数（高级项折叠）
7. Step3 资源与计划
8. Step4 复核与确认
9. 启动训练
10. 系统展示进度
11. 完成后通知

## 附件流程要求
- 文件需始终可见
- 每个文件可删除
- 状态统一：uploading / processing / ready / error

## 多步骤流程要求
- 顶部进度条 + 步骤计数
- 可前进/后退
- 支持中断恢复
